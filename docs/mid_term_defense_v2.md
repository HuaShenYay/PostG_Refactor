# 第一部分：项目背景与研究意义

## 1.1 研究背景

### 1.1.1 推荐系统概述

推荐系统（Recommender System）是解决信息过载问题的重要技术手段，通过分析用户的历史行为和偏好，主动为用户推送可能感兴趣的内容。推荐系统已广泛应用于电子商务、影视平台、音乐服务、新闻资讯等领域。

### 1.1.2 诗词推荐的特殊性

与传统商品推荐相比，诗词推荐具有以下独特挑战：

| 特点 | 描述 |
| --- | --- |
| **内容理解困难** | 诗词语言凝练含蓄，语义理解需要专业知识 |
| **用户偏好隐晦** | 用户对诗词的喜好难以通过显式评分准确表达 |
| **数据稀疏性** | 诗词领域用户交互数据相对稀疏 |
| **冷启动问题** | 新用户或新诗词难以获得有效推荐 |

### 1.1.3 研究目标

本研究旨在构建一个融合**协同过滤**与**语义相似度**的诗词推荐系统，主要目标包括：

1. **提升推荐准确度**：通过语义嵌入提取诗歌曲义特征，缓解数据稀疏问题
2. **优化用户体验**：实现多维度个性化推荐，包括热门推荐、相似推荐、个性化推荐
3. **算法创新**：提出基于显式User-CF的三路融合推荐算法

# 第三部分：核心算法详解

## 3.1 算法体系概述

本研究实现了**显式User-CF增强协同过滤算法**，融合三路推荐通道：

```
┌─────────────────────────────────────────────────────────────┐
│                    推荐算法架构                               │
├──────────────────┬───────────────────┬─────────────────────┤
│   Item-CF        │    User-CF       │     Semantic        │
│   物品协同过滤     │    用户协同过滤    │     语义相似度      │
│   (评分矩阵)      │    (显式计算)      │     (嵌入向量)      │
├──────────────────┼───────────────────┼─────────────────────┤
│ 基于用户评分行为   │ 基于用户相似度    │ 基于诗词内容语义    │
│ 计算物品相似度    │ 找到相似用户群体   │ 计算文本相似度      │
└──────────────────┴───────────────────┴─────────────────────┘
                              ↓
                    三路评分融合
```

## 3.2 算法核心：显式User-CF增强协同过滤

### 3.2.1 创新点

| 创新点 | 说明 |
| --- | --- |
| **显式User-CF** | 直接基于评分矩阵计算用户相似度，而非依赖语义画像 |
| **平衡权重** | fit阶段使用0.5×CF + 0.5×语义，平衡协同信号与语义信息 |
| **动态融合** | recommend阶段根据用户活跃度动态调整三路权重 |

### 3.2.2 算法流程

```
训练阶段 (fit)
─────────────────────────────────────────────
输入: poems[], interactions[]
     │
[1] 数据过滤
    └── 过滤低频用户/物品（交互>5000时）
     │
[2] 评分矩阵构建
    └── 构建用户-物品评分矩阵 R (m×n)
     │
[3] 语义嵌入生成
    └── SentenceTransformer: paraphrase-multilingual-MiniLM-L12-v2
    └── 输出: item_embeddings (n×384)
     │
[4] Item-CF相似度
    └── 基于评分矩阵计算物品-物品相似度
     │
[5] User-CF相似度 [新增]
    └── 基于评分矩阵计算用户-用户相似度
    └── 显式协同信号，不依赖语义
     │
[6] 相似度融合
    └── enhanced_sim = 0.5 × CF_sim + 0.5 × Semantic_sim

推荐阶段 (recommend)
─────────────────────────────────────────────
输入: user_interactions[], top_k
     │
[1] 获取用户状态
    └── user_ratings: 评分向量
    └── user_profile: 语义画像
    └── user_idx: 用户索引（用于显式UCF）
     │
[2] 动态权重选择
    └── n_rated < 5:   sem=0.80, ucf=0.15, icf=0.05
    └── 5 ≤ n < 15:   sem=0.55, ucf=0.30, icf=0.15
    └── 15 ≤ n < 30:  sem=0.40, ucf=0.35, icf=0.25
    └── n ≥ 30:       sem=0.30, ucf=0.40, icf=0.30
     │
[3] 三路评分计算
    ├── Semantic: item_embeddings @ user_profile
    ├── UCF: 显式User-CF，基于真实用户相似度
    └── ICF: enhanced_sim @ 用户已评分物品
     │
[4] 加权融合 + 排序输出
```

## 3.3 核心公式

### 3.3.1 相似度融合公式（fit阶段）

$$
S_{enhanced} = \alpha \cdot S_{CF}^{(norm)} + \beta \cdot S_{semantic}^{(norm)}
$$

其中：

- $\alpha = 0.5$（协同过滤权重）
- $\beta = 0.5$（语义相似度权重）
- 上标 $(norm)$ 表示 Min-Max 归一化

### 3.3.2 User-CF相似度计算

$$
sim_{user}(u, v) = \frac{\sum_{i \in I_{uv}}(r_{u,i} - \bar{r}_u)(r_{v,i} - \bar{r}_v)}{\sqrt{\sum_{i \in I_{uv}}(r_{u,i} - \bar{r}_u)^2} \cdot \sqrt{\sum_{i \in I_{uv}}(r_{v,i} - \bar{r}_v)^2}}
$$

其中：

- $I_{uv}$：用户u和v共同评分过的物品集合
- $\bar{r}_u$：用户u的平均评分

### 3.3.3 用户语义画像构建

$$
\vec{P}_u = \frac{\sum_{i \in R_u} w_i \cdot \vec{e}_i}{\sum_i |w_i|}
$$

其中：

- $w_i = r_{ui} - \bar{r}$（评分偏离均值）
- $\vec{e}_i$ 是物品的语义嵌入向量

### 3.3.4 User-CF评分预测

$$
score_{ucf}(u, i) = \frac{\sum_{v \in N_k(u)} sim_{user}(u, v) \cdot r_{v,i}}{\sum_{v \in N_k(u)} |sim_{user}(u, v)|}
$$

其中 $N_k(u)$ 是与用户u最相似的Top-K用户集合。

## 3.4 动态权重机制

根据用户评分数量动态调整三路权重：

| 用户活跃度 | 评分数量 | 语义权重 | UCF权重 | ICF权重 |
| --- | --- | --- | --- | --- |
| 冷启动 | <5 | 0.80 | 0.15 | 0.05 |
| 低活跃 | 5-14 | 0.55 | 0.30 | 0.15 |
| 中活跃 | 15-29 | 0.40 | 0.35 | 0.25 |
| 高活跃 | ≥30 | 0.30 | 0.40 | 0.30 |

**设计理念**：

- 用户评分少时，依赖语义相似度（因为协同数据不够）
- 用户评分多时，增加协同过滤权重（因为有足够的行为数据）

# 第五部分：算法对比分析

## 5.1 与传统算法的对比

| 算法 | 原理 | 优点 | 缺点 |
| --- | --- | --- | --- |
| 纯Item-CF | 基于评分矩阵计算物品相似度 | 协同信号强 | 冷启动差 |
| 纯Content-Based | 基于内容特征匹配 | 无冷启动 | 推荐单一 |
| 隐式User-CF | 基于语义画像计算用户相似度 | 可解释性差 | 不够直接 |
| **本研究算法** | 显式User-CF + 三路融合 | 平衡协同与语义 | 需要数据量 |

## 5.2 改进点说明

### 5.2.1 隐式 vs 显式 User-CF

| 对比项 | 隐式User-CF | 显式User-CF |
| --- | --- | --- |
| 相似度计算 | 用户语义画像的余弦相似度 | 评分矩阵的用户余弦相似度 |
| 信号来源 | 依赖物品语义嵌入质量 | 直接来自用户行为 |
| 协同信号 | 间接 | 直接 |
| 冷启动 | 依赖新用户的语义偏好 | 依赖共同评分用户数 |

### 5.2.2 权重配置改进

| 配置 | 说明 |
| --- | --- |
| 修改前 | 0.35×CF + 0.65×语义（偏向语义） |
| 修改后 | 0.5×CF + 0.5×语义（平衡） |
